# 详细使用说明

## 适用对象

- 第一次使用 Blockbench 的 Minecraft 创作者
- Java 版 Mod 作者
- 地图、服务器和资源包制作者
- 需要模型、动画、粒子、音效和 Mod 联动的独立开发者

## 启动方式

```text
$create-blockbench-minecraft-models
```

也可以直接说：

```text
使用方界造模，帮我制作一个 Minecraft 模型。
```

## 推荐首次输入

```text
我要制作一个机械风格的水晶防御塔。
黑铁材质，青蓝色能源核心，需要待机、攻击、冷却、归位和损毁动画。
先询问完整需求并生成三套概念方案，不要直接建模。
```

## 第一步：选择项目路线

说明是否已有 Mod、Minecraft 版本和加载器。如果不知道，可以描述目标玩法，让系统先帮助分类。

```text
我还没有 Mod，先制作模型。
以后准备接入 Minecraft Java 1.21.1 NeoForge。
```

## 第二步：确认保存位置

```text
保存到 D 盘，父目录使用 D:\Minecraft-Blockbench-Models。
```

系统会展示完整目标，例如：

```text
D:\Minecraft-Blockbench-Models\energy_defense\crystal_tower__v1
```

核对后明确回复“确认这个路径”。已有目录不会被静默覆盖。

## 第三步：说明主体和关联资产

```text
主体是水晶防御塔。
还需要独立能量弹、核心掉落物和损毁残骸。
不需要展示底座。
```

只要关联对象需要独立 `.bbmodel`，就属于独立模型和独立文件夹。

## 第四步：锁定设计

建议说明：

- 游戏用途和观看距离
- 模型尺寸与轮廓
- 主材质和配色
- 发光区域
- 方块化程度
- 不允许出现的结构

示例：

```text
塔高约四个方块。底座稳重，中部为开放式黑铁框架。
四个青蓝能量方块从下向上输送，顶部阶梯水晶负责攻击。
两块护盾围绕水晶公转，不能自转、重叠或接触水晶。
```

## 第五步：选择纹理精度

| 尺寸 | 适合场景 | 主要代价 |
|---:|---|---|
| 16×16 | 原版风小型资产 | 材质细节有限 |
| 32×32 | 普通物品与方块 | 适中 |
| 64×64 | 大多数高质量 Mod 模型 | 推荐起点 |
| 128×128 | Boss、大型机械、复杂武器 | 制作与显存成本更高 |
| 256×256 | 展示级大型资产 | 必须证明细节确有必要 |

示例：

```text
使用 128×128，要求黑铁划痕、边缘高光、凹槽阴影和独立发光区域，不能简单放大低清纹理。
```

## 第六步：动画和损毁

逐项说明待机、启动、移动、攻击、技能、受击、冷却、归位、死亡和损毁。

```text
攻击结束后，护盾继续环绕五秒再减速归位。
血量低于 20% 时一侧结构损坏并泄漏能量。
死亡后切换到独立残骸模型。
```

## 第七步：粒子和音频

粒子需要说明位置、方向、颜色、数量、生命周期和触发事件。

音频文件可以直接拖入对话框，中文名称或编号均可。系统会先生成英文事件映射，确认后再处理。

## 第八步：确认三套概念图

检查生图前需求表后回答：

```text
需求正确，可以生成三套概念方案。
```

选择时可以说：

```text
选择第二套。保留第二套主体，但把水晶改成第一套的结构。
重新生成最终确认图，不要开始正式建模。
```

## 第九步：批准正式生产

最终方案满意后明确输入：

```text
确认这个方案，开始生成正式 Blockbench 模型。
```

“我比较喜欢第二个”只表示偏好，不等于正式生产批准。

## 第十步：反馈修改

推荐格式：

```text
对象：左侧护盾
位置：顶部水晶左侧
问题：公转时距离水晶太近
目标：旋转半径增加三个 Blockbench 单位
约束：两块护盾保持相同方向，不允许自身旋转
```

系统应先定位问题，再做单一目标修改，避免连带改变其他结构。

## 第十一步：Mod 联动

提供真实 Mod 项目路径并批准接入。系统会检查版本、加载器、JDK、Gradle、Mod ID 和资源结构，再处理注册、动画控制、投射物、粒子、音效和游戏逻辑。

## 常用场景

### 只做模型

```text
我暂时没有 Mod，只制作完整的 bbmodel、纹理和动画。
```

### 修改已有模型

```text
检查我上传的 bbmodel。不要重做，先诊断骨骼原点、动画衔接和穿模，再展示修改方案。
```

### 一组关联资产

```text
制作主体 Boss、独立投射物、核心掉落物和损毁残骸。每个模型使用独立文件夹，音效和动画不能混绑。
```

## 用户需要记住的四句话

1. 先说目标，不必先学会软件术语。
2. 看清需求表，再批准生成概念图。
3. 选择方案不等于批准正式建模。
4. 最终交付看验证证据，不只看“文件已生成”。
